Course Map

MATH 6361: Mathematical Logic.

A graduate-level online course organized as a sequence of modules in proof, semantics, soundness, completeness, quantification, set theory, and incompleteness. This page turns the Blackboard structure into a public map of the course architecture.

How the Course Works

Each module asks students to move through guiding questions, learning outcomes, reading notes, video notes, homework, discussion, and quiz practice. The rhythm is deliberate: read, watch, formalize, prove, reflect.

Modules 0-5

From atomic sentences to Boolean proof.

Module 0: Atomic Sentences

Students begin with constants, predicates, function symbols, terms, and the discipline of translating ordinary language into a first-order language.

  • Identify terms, complex terms, predicates, and atomic sentences.
  • Translate between colloquial English and first-order atomic sentences.
  • Compare the roles of constants, variables, predicates, and function symbols.
Core schema: an atomic sentence has the form \(P(t_1,\ldots,t_n)\), where each \(t_i\) is a term.
YouTube lecture slot: Atomic Sentences

Modules 1-3: Propositional Logic and Boolean Connectives

The course then stabilizes validity, soundness, tautology, logical consequence, and truth-table reasoning.

  • Classify arguments as valid/invalid and sound/unsound.
  • Construct and interpret truth tables for \(\neg\), \(\wedge\), \(\vee\), \(\to\), and \(\leftrightarrow\).
  • Distinguish logical truth, tautology, logical equivalence, and tautological equivalence.
Example question: when does \(\Gamma \models \varphi\) agree with truth-table consequence?
YouTube lecture slots: Propositional Logic, Boolean Connectives, Tautologies

Modules 4-5: Formal Proof Methods

Students move from semantic tests to formal derivations, including subproofs, contradiction, cases, conditionals, and biconditionals.

  • Use introduction and elimination rules in Fitch-style proof.
  • Formalize proof by contradiction and proof by cases.
  • Translate English conditionals into standard logical form.
Core habit: every line in a proof should be justified by a rule, a premise, or the scope of a subproof.
YouTube lecture slots: Boolean Proofs and Conditionals

Modules 6-7

Soundness, completeness, and review.

Module 6: Soundness and Completeness

This module makes the central bridge explicit: syntactic derivability and semantic consequence are different ideas, but a well-behaved proof system relates them.

  • State soundness: if \(\Gamma \vdash \varphi\), then \(\Gamma \models \varphi\).
  • State completeness: if \(\Gamma \models \varphi\), then \(\Gamma \vdash \varphi\).
  • Use induction as a proof method in metatheoretic arguments.
The module asks students to see proof systems as mathematical objects rather than just procedures.
YouTube lecture slot: Soundness and Completeness

Module 7: Midterm Review

The review module helps students reorganize the first half of the course around concepts, not just homework problem numbers.

  • Review syntax, semantics, translation, truth tables, and proof rules.
  • Identify which technique is appropriate for a given kind of logic problem.
  • Prepare for proof-writing under exam conditions.

Modules 9-11

Quantification and first-order proof.

Module 9: The Logic of Quantification

Students extend the propositional toolkit to variables, quantifiers, free and bound occurrence, and truth in first-order structures.

  • Recognize well-formed formulas and sentences.
  • Identify free and bound variables.
  • Determine truth values of quantified sentences in a structure.
Core contrast: \(\forall x\,P(x)\) asserts universal coverage, while \(\exists x\,P(x)\) asserts witness existence.
YouTube lecture slot: Introduction to Quantification

Modules 10-11: Proof Methods for Quantifiers

Students learn both informal and formal proof methods for quantified statements, including universal and existential introduction/elimination.

  • Translate English sentences containing multiple quantifiers.
  • Recognize how quantifier order changes meaning.
  • Construct Fitch proofs using quantifier rules and identity.
Example contrast: \(\forall x\exists y\,R(x,y)\) and \(\exists y\forall x\,R(x,y)\) usually say very different things.
YouTube lecture slots: Quantifier Proofs

Module 12

Axiomatic set theory and mathematical structures.

Module 12: Sets, Relations, and Structures

The course uses set theory as a natural setting for first-order reasoning: extension, separation, power sets, unions, intersections, relations, functions, and structures.

  • Apply the axiom of extension and the schema of separation.
  • Work with \(\mathcal{P}(A)\), \(\bigcup A\), intersections, relations, and functions.
  • Describe mathematical structures using sets, relations, functions, and constants.
Set theory becomes a language for saying what mathematical worlds are made of.
YouTube lecture slot: Axiomatic Set Theory

Modules 13-14

Completeness and incompleteness.

Final Modules

The course ends by placing first-order completeness beside incompleteness, so students can see both the power and the limits of formal systems.

  • State the soundness theorem for first-order logic.
  • State the completeness theorem for first-order logic.
  • State the incompleteness theorem for Peano Arithmetic.
  • Explain why formal proof and mathematical truth do not collapse into the same concept in sufficiently expressive systems.
Final tension: first-order logic is complete, but arithmetic powerful enough to encode syntax is not complete in the same way.
