Coursebook

MATH 6361: Mathematical Logic

A graduate-level online course in formal languages, proof, semantics, soundness, completeness, quantification, set theory, and incompleteness. This page presents the Blackboard module sequence as a public course map.

All Courses LaTeX Module Summary

1. Syntax and Atomic Sentences

The course begins by building a formal language from constants, variables, function symbols, predicate symbols, terms, and atomic sentences. Students learn that formal notation is a designed language with grammar.

Objectives

  • Identify terms and well-formed atomic sentences.
  • Translate simple English statements into first-order notation.
  • Distinguish constants, variables, predicates, and functions.

Module

Module 0 introduces atomic sentences and the basic syntax of first-order languages.

Representative example: decide whether \(P(f(a),x)\) is a term, formula, or sentence, then explain what each symbol is doing.

2. Truth Tables, Validity, and Logical Consequence

Students next study propositional connectives, truth tables, tautologies, logical equivalence, validity, soundness of arguments, and semantic consequence.

Objectives

  • Classify arguments as valid or invalid.
  • Build truth tables for Boolean connectives.
  • Distinguish truth, tautology, equivalence, and consequence.

Modules

Modules 1-3 develop propositional logic, Boolean connectives, tautologies, and semantic tests.

Representative example: use a truth table to test whether \((P\to Q)\) and \((\neg P\vee Q)\) are equivalent.

3. Formal Proof Methods

The course then shifts from semantics to derivations. Students learn introduction and elimination rules, subproofs, conditionals, biconditionals, contradiction, and proof by cases.

Objectives

  • Construct Fitch-style formal proofs.
  • Use subproof scope correctly.
  • Choose rules that match the main connective of the target formula.

Modules

Modules 4-5 focus on Boolean proof and conditional proof methods.

Representative example: prove \(P\to R\) from \(P\to Q\) and \(Q\to R\) using conditional proof.

4. Soundness, Completeness, and Review

Students step back from doing proofs to proving facts about proof systems. Soundness connects derivability to truth preservation; completeness connects semantic consequence back to derivability.

Objectives

  • State soundness and completeness precisely.
  • Explain the difference between \(\Gamma\vdash\varphi\) and \(\Gamma\models\varphi\).
  • Use induction in metatheoretic arguments.

Modules

Modules 6-7 consolidate the first half of the course and introduce metatheoretic proof.

Representative example: explain why soundness says no formal proof system should prove a false conclusion from true premises.

5. Quantification and First-Order Proof

Variables and quantifiers bring the language closer to ordinary mathematics. Students study free and bound variables, truth in structures, universal and existential statements, and proof rules for quantifiers.

Objectives

  • Recognize free and bound variables.
  • Translate statements with nested quantifiers.
  • Construct proofs using quantifier introduction and elimination rules.

Modules

Modules 9-11 develop the logic of quantification and formal proof methods for quantified statements.

Representative example: compare \(\forall x\exists y\,R(x,y)\) and \(\exists y\forall x\,R(x,y)\) by giving an interpretation where one is true and the other false.

6. Set Theory, Relations, Functions, and Structures

Set theory gives the course a natural mathematical universe. Students use extension, separation, power sets, unions, intersections, relations, functions, and structures to formalize mathematical objects.

Objectives

  • Apply basic axioms and constructions of set theory.
  • Work with relations and functions as sets.
  • Describe structures in a first-order language.

Module

Module 12 uses set theory as a setting for first-order reasoning.

Representative example: describe a graph as a structure with a domain and a binary relation.

7. Completeness and Incompleteness

The final modules place first-order completeness beside incompleteness. Students see both the power of formal proof and the limits of formal systems expressive enough to encode arithmetic.

Objectives

  • State first-order soundness and completeness.
  • State the incompleteness theorem for Peano Arithmetic.
  • Explain why truth and provability diverge in sufficiently expressive systems.

Modules

Modules 13-14 close with completeness, incompleteness, and the philosophical consequences for foundations.

Representative example: explain how first-order logic can be complete while arithmetic is incomplete.
